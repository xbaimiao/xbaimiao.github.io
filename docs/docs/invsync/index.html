介绍

前言​

在使用HuskSync和其他同步插件时，我们发现无法同步某些附魔和物品,玩家背包有可能丢失 。这给我们的服务器管理带来了很大的困扰。因此，我们开发了InvSync来解决这个问题。

与其他同步插件不同的是，InvSync支持同步更多的附魔和物品，避免了数据丢失和不一致的问题。

支持版本1.12.2-1.20.2 Folia服务端也支持

支持同步的数据​

  • statistic (统计数据)
  • exp (经验与等级)
  • ender (末影箱)
  • inventory (背包)
  • health (血量)
  • food (饥饿值)
  • advancement (成就)
  • buff (药水效果)
  • maxHealth (最大血量) 默认关闭
  • persistentDataContainer (玩家PDC持久化数据) 1.14.1+才有 默认关闭

以下需要版本 2.0.0 +​

  • gameMode (游戏模式) 默认关闭
  • op (是否为管理员) 默认关闭

Mod数据​

  • 像素宝可梦(Pixelmon) 世代 默认关闭
  • 像素宝可梦(Pixelmon) 重铸 默认关闭

介绍​

  1. InvSync是一款高效的跨服同步插件，它可以让您在多个服务器之间同步玩家背包、物品、经验等数据。
  2. InvSync采用了redis + mysql的储存方式，确保数据的安全性和实时性。
  3. InvSync易于配置和使用，可以轻松实现跨服同步，提高服务器的玩家体验和管理效率。
  4. InvSync还支持从HuskSync进行数据迁移，因此您可以轻松地切换到InvSync而无需担心玩家数据丢失
  5. 对于在线玩家，您可以使用其他插件如CMI来编辑他们的背包数据。对于离线玩家，您可以**使用InvSync的编辑功能来修改他们的背包和末影箱数据 **。

你问我答​

从KnapsackToGo4迁移​

先不要动KnapsackToGo4 插件和配置都别动
关闭所有正在运行的子服 选择其中一个子服 安装好InvSync 
InvSync需要确保数据库 redis 链接正常
运行命令 /invsync migrator ktg4 all
等待迁移完成 关服删除KnapsackToGo4即可

从MysqlPlayerDataBridge迁移​

先不要动MysqlPlayerDataBridge 插件和配置都别动
关闭所有正在运行的子服 选择其中一个子服 安装好InvSync 
InvSync需要确保数据库 redis 链接正常
运行命令 /invsync migrator mysqlplayerdatabridge all
等待迁移完成 关服删除MysqlPlayerDataBridge即可

从HuskSync迁移​

插件版本 1.2.5 以及以上适用

先不要动HuskSync 插件和配置都别动
关闭所有正在运行的子服 选择其中一个子服 安装好InvSync 
InvSync需要确保数据库 redis 链接正常
运行命令 /invsync migrator husksync all
等待迁移完成 关服删除HuskSync即可

插件版本 1.2.5 以下适用

关闭所有正在运行的子服 选择其中一个子服 安装好InvSync 和我修改过的HuskSync
HuskSync和InvSync都需要配置好数据库和redis 并确保能连接
运行命令 /invsync migrator husksync all
等待迁移完成 然后将所有子服的HuskSync卸载只保留InvSync即可
我修改的HuskSync将会在你购买InvSync后一并发送给你

关于数据加载超时​

此问题已在1.2.3版本解决，需要在bc端也安装InvSync 只需配置redis链接

玩家 可能会被服务器禁止进入服务器 踢出消息是 “§c加载数据超时” 
这是因为玩家所在的服务器 非正常关闭（蹦服，或强制关闭）
造成了这个服务器内的所有玩家数据锁定了，
需要你手动执行命令 /invsync lock <player> false 给他解锁 他就可以进了
如果玩家太多 可直接/invsync unlockAll 解锁所有玩家数据

PAPI变量​

  • %invsync_servername% 获取指定服务器在线玩家数量
  • %invsync_totalOnline% 获取当前BC在线玩家数量

内置文件​

配置文件​

# 调试模式（开启此选项后台日志会变多）
debug: false
redis:
  host: localhost
  port: 6379
  #  password: ""
mysql:
  host: 'localhost'
  port: 3306
  database: 'database'
  user: 'user'
  passwd: 'passwd'
  ssl: false
mobs:
  # 是否同步宝可梦数据
  pixelmon: false
sync:
  # 是否同步统计数据
  statistics: true
  # 是否同步经验等级
  exp: true
  # 是否同步末影箱
  ender: true
  # 是否同步血量
  health: true
  # 是否同步饥饿值
  food: true
  # 是否同步背包
  inventory: true
  # 是否同步成就数据
  advancement: true
  # 是否同步药水效果
  buff: true
  # 是否同步游戏模式
  gameMode: false
  # 是否同步最大血量
  maxHealth: false
  # 是否同步PersistentDataContainer数据
  persistentDataContainer: false
  # 是否同步OP
  op: false
# 同步时跳过的物品名称 名称会忽略颜色符号完全匹配
skip-items:
  - '不同步物品名1'
  - '不同步物品名2'
# 数据快照
snapshot:
  # 是否启用快照保存
  enable: false
  # 启用的快照保存原因 未启用则在这个原因触发时 不会保存数据快照
  # DISCONNECT("断开连接"), WORLD_SAVE("保存世界"), COMMAND("命令"), SHUT_DOWN("服务器关闭"), AUTO_SAVE("自动保存")
  enableReason:
    - 'DISCONNECT'
    - 'WORLD_SAVE'
    - 'COMMAND'
    - 'SHUT_DOWN'
    - 'AUTO_SAVE'
  # 单个玩家 最大置顶快照数量 超过会自动删除最早的
  maxPined: 10
  # 单个玩家 最大普通快照数量 超过会自动删除最早的
  maxNormal: 20
  # 自动保存快照的时间 这个快照会自动置顶 时间单位可用 d h m s ms
  autoSave: '30m'
